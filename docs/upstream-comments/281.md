Encountered the same pattern + reproduced root cause in our fork [`opnd-io/opnd-codex-plugin`](https://github.com/opnd-io/opnd-codex-plugin) (PR #4 commit [`3d72d6e`](https://github.com/opnd-io/opnd-codex-plugin/commit/3d72d6e)).

## Root cause (2-factor)

1. **Plugin home isolation (v2.0+)**: plugin sessions in `$HOME/.codex/claude-code/` — `codex logout && codex login` only updates `~/.codex/auth.json`; plugin home stays stale.
2. **Broker daemon stale cache**: shared broker pipe holds invalidated session, doesn't re-read auth.json after the underlying file changes.

## Plugin-side fix

`plugins/opnd-codex/scripts/lib/codex.mjs` `getCodexAuthStatusFromClient` catch now branches:

- `error.rpcCode === BROKER_BUSY_RPC_CODE` (-32001) → returns `loggedIn: null + transient: true`
- regex `/timed out|timeout|ECONNRESET|EPIPE/i` → same transient branch (broker stuck case)

Callers (incl. `setup --json`) distinguish *transient broker contention* from *actual logged-out* state — no more false-negative `loggedIn: false` while the user is actually authenticated.

## Recovery procedure (cross-platform)

```bash
# 1. Sync plugin home auth from root
cp ~/.codex/auth.json ~/.codex/claude-code/auth.json

# 2. Kill stale brokers (Windows PowerShell — case-sensitive lowercase only, excludes Codex Desktop GUI)
Get-Process | Where-Object { $_.ProcessName -ceq 'codex' } | Stop-Process -Force

# macOS / Linux equivalent
pkill -f 'codex.*app-server'

# 3. If broker stuck (large plugin home SQLite WAL >10MB)
rm ~/.codex/claude-code/*.sqlite-wal ~/.codex/claude-code/*.sqlite-shm

# 4. Verify
node plugins/opnd-codex/scripts/codex-companion.mjs setup --json
```
