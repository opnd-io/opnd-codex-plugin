Our fork [`opnd-io/opnd-codex-plugin`](https://github.com/opnd-io/opnd-codex-plugin) addresses this in the v2.0+ sprint using `cmd.exe` wrapping (avoiding the `shell: true` command-injection risk):

## Fix location

`plugins/opnd-codex/scripts/lib/process.mjs` L62-82 — `buildCommandInvocation()` helper:

- Detects `.cmd` / `.bat` shims on Windows (incl. `codex.cmd` via npm global)
- Wraps invocation as `cmd.exe /d /s /c call ...`
- Keeps `shell: false`
- Escapes each argv via `quoteWindowsCmdArg()`

This is safer than `shell: true` (which passes the entire command string through the shell parser → injection risk for any argv containing shell metacharacters) while still resolving PATHEXT.

`scripts/lib/app-server.mjs` L573 invokes the helper for `spawn("codex", ...)`. Same pattern can be applied to other `spawn()` sites.
