Our fork [`opnd-io/opnd-codex-plugin`](https://github.com/opnd-io/opnd-codex-plugin) ships a `--max-findings <N>` flag on `/opnd-codex:review`:

- Default: 20
- Hard cap: 100
- Example: `/opnd-codex:review --max-findings 50` (lifts the implicit 2-3 cap on large diffs)

See the [`/opnd-codex:review` section in README](https://github.com/opnd-io/opnd-codex-plugin/blob/main/README.md#opnd-codexreview) for the documented behavior. Implementation in `plugins/opnd-codex/scripts/codex-companion.mjs` review handler.

Cross-reference [`docs/exploration/2026-05-28-upstream-issue-classification.md`](https://github.com/opnd-io/opnd-codex-plugin/blob/main/docs/exploration/2026-05-28-upstream-issue-classification.md). Happy to send a PR if upstream wants to adopt the same flag shape.
