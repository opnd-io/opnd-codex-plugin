## Fix location

`plugins/opnd-codex/scripts/lib/process.mjs` L62-82 — wraps `.cmd` / `.bat` invocations (which Windows needs PATHEXT resolution for) with `cmd.exe /d /s /c call ...` while keeping `shell: false` and escaping each argv via `quoteWindowsCmdArg()`.

This avoids both the ENOENT failure (Node not trying PATHEXT for shims) and the command-injection risk of `shell: true`.

## Usage

`scripts/lib/app-server.mjs` L573 invokes the helper. Same pattern can be applied to any `spawn("codex", ...)` site upstream.
