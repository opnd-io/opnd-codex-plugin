Our fork [`opnd-io/opnd-codex-plugin`](https://github.com/opnd-io/opnd-codex-plugin) has implemented this in the v2.0+ sprint as a **BREAKING change** with an opt-out:

## Implementation

- Plugin sessions now land in `$HOME/.codex/claude-code/` instead of `~/.codex/` — separate `auth.json`, `config.toml`, `state.sqlite`, `logs.sqlite`, `sessions/`.
- Legacy shared home restoration: `CODEX_PLUGIN_USE_DEFAULT_HOME=1` env var.

## Trade-offs observed (please note before adopting)

The isolation creates a known second-order issue: when users run `codex logout && codex login`, only the root home auth refreshes — the plugin home stays stale. Our fork addresses this in PR #4 ([`3d72d6e`](https://github.com/opnd-io/opnd-codex-plugin/commit/3d72d6e)) by adding broker-busy/timeout branches in `getCodexAuthStatusFromClient` so callers can distinguish *transient* from *actual logged-out*. See also #281 / #342 for context.

If upstream adopts the isolation, recommend:
- Surface the `cp ~/.codex/auth.json ~/.codex/claude-code/auth.json` recovery step in `commands/setup.md` disclaimer
- Add `inspectPluginHomeFreshness()` (mtime compare + WAL size detect) to `buildSetupReport` next-steps
- Cross-platform recovery hint (Windows PowerShell + macOS/Linux pkill)
